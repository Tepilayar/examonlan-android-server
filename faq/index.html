Pusat Bantuan & FAQ Ujian

Cari solusi cepat untuk mengatasi kendala teknis aplikasi Examonlan di HP Android, iOS, maupun Laptop PC.

Demi menjaga integritas ujian dan mencegah kecurangan (seperti menyalin layar, memanipulasi lokasi GPS, kontrol jarak jauh, dll.), aplikasi Examonlan mewajibkan Opsi Developer (Opsi Pengembang) dan Mendebug USB (USB Debugging) dimatikan sepenuhnya.

Langkah Menonaktifkan Opsi Pengembang (Umum):

  1. Buka menu Setelan / Pengaturan (Settings) pada HP Android Anda.
  2. Gulir ke bawah dan cari menu Pengaturan Tambahan (Additional Settings) atau Sistem (System).
  3. Temukan menu Opsi Pengembang / Opsi Developer (Developer Options).
  4. Geser tombol toggle di bagian atas ke posisi Nonaktif (Mati / OFF).
  5. Buka kembali aplikasi Examonlan.

Panduan Khusus per Merk HP:

  • Xiaomi / Redmi / POCO: Buka Pengaturan → Setelan Tambahan → Opsi Pengembang → Geser toggle ke OFF.
  • Oppo / Realme: Buka Pengaturan → Pengaturan Tambahan → Opsi Developer → Geser toggle ke OFF.
  • Samsung: Buka Pengaturan → Pilihan Pengembang (di bagian paling bawah) → Geser toggle ke OFF.
  • Vivo: Buka Pengaturan → Pengelolaan Sistem → Opsi Developer → Geser toggle ke OFF.
  • Infinix: Buka Pengaturan → Sistem → Opsi Pengembang → Geser toggle ke OFF.
Jika Anda kesulitan menemukan menu ini, Anda juga bisa mengetik "Opsi Pengembang" di kolom pencarian pada menu Pengaturan HP Anda.

Aplikasi Examonlan mendeteksi aplikasi yang dapat mengambang di atas layar (draw over other apps) yang sering digunakan untuk menyontek atau merekam layar. Contohnya adalah Game Booster / Game Turbo bawaan HP, perekam layar (Screen Recorder), aplikasi translate melayang, assistive touch, serta aplikasi ganda (cloner apps).

Cara Mengatasinya:

  1. Tutup semua aplikasi yang sedang aktif melalui tombol Recent Apps (tombol kotak di bagian bawah layar) lalu bersihkan semuanya.
  2. Nonaktifkan fitur Game Turbo / Game Space / Game Booster bawaan HP Anda terlebih dahulu sebelum memulai ujian.
  3. Jika Anda mengaktifkan Aplikasi Ganda (Dual Apps) untuk aplikasi ujian, silakan hapus atau nonaktifkan fitur kloningan tersebut.
  4. Buka kembali aplikasi Examonlan.

Masalah layar hitam biasanya disebabkan oleh kegagalan aplikasi dalam memuat Google Form ujian Anda. Hal ini umumnya disebabkan oleh dua faktor utama:

Langkah Penanganan:

  1. Koneksi Internet Lambat/Putus: Pastikan HP Anda terhubung ke jaringan internet (Wi-Fi sekolah or paket data) yang stabil dan cepat.
  2. Android System WebView Usang: Examonlan menggunakan mesin internal Chrome untuk memuat Google Form. Jika webview di HP Anda sudah terlalu lama tidak diperbarui, halaman Google Form akan gagal dimuat.

Cara Memperbarui Android System WebView:

  1. Buka aplikasi Google Play Store di HP Anda.
  2. Ketik dan cari "Android System WebView".
  3. Klik tombol Update (Perbarui) dan tunggu hingga selesai.
  4. Disarankan juga untuk memperbarui aplikasi Google Chrome ke versi terbaru.
  5. Restart HP Anda, kemudian buka kembali Examonlan.

Token Ujian di Examonlan terikat secara ketat dengan parameter waktu server sekolah. Jika Anda mengalami error ini, ikuti petunjuk berikut:

Solusi:

  1. Atur Waktu HP ke Otomatis: Masuk ke Pengaturan HP → Tanggal & Waktu → Aktifkan "Gunakan Waktu Jaringan / Atur Otomatis". Jika waktu HP Anda selisih lebih dari 1 menit dengan server, token akan dianggap tidak valid.
  2. Periksa Kembali Token: Pastikan Anda memasukkan kode token ujian 6 digit dengan benar dan tidak tertukar antara angka dan huruf (misal angka 0 dan huruf O, atau angka 1 dan huruf l).
  3. Waktu Ujian Belum Dimulai: Hubungi guru Anda untuk menanyakan apakah jadwal ujian tersebut sudah diaktifkan di dashboard sekolah.

Examonlan didesain untuk mengunci perangkat siswa di layar ujian (lockdown mode) agar siswa tidak bisa kabur ke aplikasi lain. Untuk keluar secara aman setelah selesai ujian:

Langkah Keluar:

  1. Pastikan siswa sudah menekan tombol Kirim (Submit) di halaman Google Form ujian.
  2. Mintalah Password Keluar kepada Guru atau Pengawas Ujian Anda.
  3. Klik tombol ikon menu atau titik tiga di pojok kanan atas aplikasi, lalu pilih Keluar / Exit.
  4. Masukkan Password Keluar yang valid untuk membuka kembali HP Anda ke mode normal.
Guru/Sekolah dihimbau untuk memberikan Password Keluar hanya ketika siswa telah dikonfirmasi menyelesaikan ujiannya.

Seluruh jawaban siswa disimpan secara langsung ke dalam formulir **Google Form** sekolah Anda (dan akan masuk otomatis ke Google Spreadsheet yang terhubung).

Examonlan tidak menyimpan atau mengintersepsi jawaban siswa di server kami demi keamanan data sekolah Anda. Kami hanya berfungsi sebagai pengaman jalannya browser ujian (exambrowser) pada perangkat siswa agar bebas dari kecurangan. Anda dapat mengakses hasil rekap ujian langsung melalui Google Drive/Google Forms akun sekolah Anda seperti biasa.

Ya, Examonlan dirancang multiplatform untuk memudahkan sekolah menyelenggarakan ujian dengan perangkat apa pun yang dimiliki siswa:

  • iOS (iPhone / iPad): Ujian diamankan menggunakan integrasi dengan aplikasi pihak ketiga resmi Safe Exam Browser (SEB). Konfigurasi file `.seb` otomatis dapat diunduh langsung melalui Dashboard sekolah Anda dan dikirim ke siswa.
  • Laptop / PC (Windows): Gunakan aplikasi desktop **Examonlan Windows** yang mengunci layar komputer sepenuhnya. Installer aplikasi Windows dapat diunduh di halaman utama (Google Drive resmi).

Silakan buka halaman beranda dan cari bagian unduhan spesifikasi untuk mendapatkan link file konfigurasi & installer desktop terbaru.

Examonlan mendeteksi adanya notifikasi atau popup dari aplikasi lain yang muncul mengganggu layar ujian. Hal ini dianggap potensi kecurangan karena dapat menampilkan informasi dari luar.

Cara Mengatasi:

  1. Aktifkan Mode Jangan Ganggu (Do Not Disturb):Buka Pengaturan HP → Suara / Sound → Jangan Ganggu (Do Not Disturb) → Aktifkan. Mode ini akan memblokir semua notifikasi selama ujian berlangsung.
  2. Matikan Notifikasi Heads-Up / Banner:Buka Pengaturan → Notifikasi → Pilih aplikasi yang sering mengirim notifikasi (WhatsApp, SMS, dll.) → Matikan tampilan "Pop on screen" atau "Notifikasi Heads-Up".
  3. Tutup semua aplikasi latar belakang melalui tombol Recent Apps sebelum memulai ujian.
  4. Buka kembali aplikasi Examonlan dan masukkan token ujian.
Pastikan tidak ada aplikasi yang aktif di latar belakang yang berpotensi mengirim notifikasi selama sesi ujian.

Saat aplikasi Examonlan dimulai, Android akan menampilkan alert konfirmasi Sematkan Layar (Screen Pinning). Siswa wajib menekan tombol OK pada alert tersebut agar layar terkunci di aplikasi ujian.

Jika siswa tidak menekan OK atau menutup alert tersebut, sistem akan mendeteksinya sebagai pelanggaran dan alarm pelanggaran akan berbunyi.

Langkah yang benar saat memulai ujian:

  1. Buka aplikasi Examonlan dan masukkan token ujian.
  2. Saat muncul pop-up "Sematkan Layar?" dari Android, segera tekan OK / Sematkan.
  3. Layar akan terkunci di dalam aplikasi ujian dan ujian dapat dimulai.

Jika alarm pelanggaran sudah terlanjur muncul:

  1. Hubungi Guru / Pengawas Ujian untuk mereset atau mengkonfirmasi kondisi perangkat siswa.
  2. Tutup aplikasi, buka kembali, dan pastikan menekan OK saat alert Screen Pinning muncul.

Ya, Examonlan mewajibkan penggunaan Google Keyboard (Gboard) sebagai keyboard default selama ujian. Keyboard pihak ketiga seperti SwiftKey, Samsung Keyboard berbasis AI, atau keyboard yang memiliki fitur sinkronisasi cloud dilarang digunakan karena berpotensi dimanfaatkan untuk kecurangan.

Cara Mengganti ke Google Keyboard (Gboard):

  1. Pastikan aplikasi Gboard sudah terpasang. Unduh dari Google Play Store jika belum ada.
  2. Buka Pengaturan HP → Manajemen Umum atau Bahasa & Input.
  3. Pilih Keyboard Default / Metode Input.
  4. Pilih Gboard sebagai keyboard utama.
  5. Buka kembali aplikasi Examonlan.
Gboard adalah keyboard resmi dari Google, aman, dan tidak memiliki fitur sinkronisasi yang dapat membocorkan jawaban ujian. Pastikan Gboard sudah diperbarui ke versi terbaru melalui Play Store.

Aplikasi desktop Examonlan saat ini hanya tersedia untuk Windows. Belum ada versi native Examonlan untuk macOS.

  • Windows (7 / 8 / 10 / 11): Gunakan aplikasi Examonlan Windows yang dapat diunduh melalui Google Drive resmi di halaman beranda. Aplikasi ini akan mengunci layar komputer sepenuhnya selama ujian berlangsung.
  • macOS (Mac / MacBook): Siswa pengguna Mac wajib menggunakan aplikasi Safe Exam Browser (SEB) sebagai pengganti. File konfigurasi .seb yang sudah disesuaikan dengan ujian sekolah dapat diunduh melalui Dashboard Guru/Sekolah.

Cara Menggunakan SEB di macOS:

  1. Unduh dan pasang Safe Exam Browser dari situs resmi safeexambrowser.org.
  2. Minta file konfigurasi .seb dari Guru atau unduh langsung melalui Dashboard sekolah.
  3. Buka file .seb tersebut, SEB akan otomatis terbuka dan mengunci layar Mac ke halaman ujian.
Dukungan platform macOS native untuk Examonlan direncanakan di pembaruan mendatang. Pantau pengumuman resmi di halaman beranda.

Semua soal ujian di Examonlan dibuat melalui Google Form. Examonlan tidak menyediakan fitur pembuatan soal sendiri — kami berfungsi sebagai pengaman browser agar siswa tidak bisa keluar dari halaman ujian.

Cara Membuat Soal Ujian:

  1. Buka Google Forms melalui forms.google.com menggunakan akun Google sekolah Anda.
  2. Buat formulir baru dan tambahkan soal-soal ujian (pilihan ganda, esai, dll.).
  3. Setelah soal selesai, salin URL/Link Google Form tersebut.
  4. Paste link tersebut ke dalam menu Buat Ujian di Dashboard Admin Examonlan sekolah Anda.
  5. Atur jadwal ujian, lalu bagikan token kepada siswa.
Pastikan Google Form diatur agar memerlukan login akun Google sebelum mengisi, sehingga jawaban siswa tersimpan dengan identitas yang jelas.

Ya, siswa wajib login dengan akun Gmail (Google) sebelum mengerjakan ujian. Ini adalah syarat dari Google Form, bukan dari Examonlan.

  • Tanpa login Gmail: Jawaban siswa tidak akan tersimpan ke Google Spreadsheet secara teridentifikasi. Guru tidak dapat mengetahui siapa yang mengerjakan soal tersebut.
  • Dengan login Gmail: Setiap jawaban otomatis tersimpan beserta identitas akun Google siswa, sehingga rekap nilai menjadi akurat dan terpercaya.

Jika siswa belum login Gmail:

  1. Minta siswa keluar dulu dari aplikasi Examonlan (hubungi guru untuk password keluar).
  2. Buka aplikasi Gmail atau Google dan login dengan akun Gmail siswa.
  3. Buka kembali Examonlan dan masukkan token ujian.
Disarankan siswa sudah login Gmail sebelum ujian dimulai agar tidak membuang waktu dan menghindari pelanggaran saat proses login.

Examonlan menyediakan dua cara pembuatan token ujian untuk mengakomodasi berbagai kondisi jaringan di sekolah:

  • Mode Online: Token ujian dibuat melalui Dashboard Admin Examonlan yang diakses via browser di komputer/laptop guru. Guru perlu terhubung ke internet untuk membuat dan mendistribusikan token. Mode ini direkomendasikan untuk penggunaan standar.
  • Mode Offline: Token ujian dibuat langsung dari smartphone guru menggunakan fitur generate token lokal, tanpa memerlukan koneksi internet untuk proses pembuatan tokennya. Cocok digunakan saat jaringan internet di sekolah sedang bermasalah.
Penting: Meskipun token dapat dibuat secara offline, pengerjaan ujian oleh siswa tetap memerlukan koneksi internet karena soal ujian dimuat dari Google Form secara online.

Pengguna iPhone dan iPad tidak dapat menggunakan aplikasi Examonlan Android. Sebagai gantinya, ujian dilakukan menggunakan Safe Exam Browser (SEB) yang tersedia di App Store secara gratis.

Langkah-langkah Ujian di iPhone / iPad:

  1. Buka App Store di iPhone/iPad Anda, cari dan install "Safe Exam Browser" (developer: ETH Zurich).
  2. Minta file konfigurasi .seb dari Guru atau Pengawas Ujian. File ini bisa diunduh langsung dari Dashboard Sekolah atau dikirim melalui WhatsApp/email.
  3. Buka file .seb tersebut dari aplikasi Files, WhatsApp, atau email — pilih "Open in Safe Exam Browser".
  4. SEB akan otomatis terbuka dan mengunci layar iPhone ke halaman ujian Google Form. Siswa tidak dapat keluar ke aplikasi lain selama ujian berlangsung.
  5. Kerjakan soal ujian, lalu tekan Submit / Kirim di halaman Google Form setelah selesai.
  6. Untuk keluar dari SEB, masukkan Password Keluar yang diberikan oleh guru melalui menu di pojok kanan atas.
Pastikan siswa sudah login Gmail di iPhone sebelum ujian dimulai agar jawaban tersimpan dengan benar di Google Form.
File .seb bersifat spesifik per ujian dan hanya berlaku selama jadwal ujian aktif. Hubungi guru jika file konfigurasi belum diterima.

Examonlan menggunakan tiga jenis PIN yang memiliki fungsi berbeda untuk menjaga keamanan dan integritas ujian:

  • PIN Masuk (Token Ujian): Token 6 digit yang dibagikan guru kepada siswa. Token ini hanya bisa digunakan sekali — setelah dipakai masuk ke halaman ujian, token tersebut tidak dapat digunakan kembali oleh perangkat yang sama.
  • PIN Pengawas: Jika siswa perlu masuk kembali ke halaman ujian (misalnya setelah HP restart atau keluar tidak sengaja), dapat menggunakan PIN Pengawas. PIN ini bersifat rahasia dan hanya boleh diinput oleh guru atau pengawas ujian, bukan oleh siswa sendiri. Guru mendatangi siswa dan menginput PIN secara langsung di perangkat siswa.
  • PIN Keluar: Digunakan untuk keluar dari aplikasi Examonlan setelah ujian selesai. Siswa meminta PIN Keluar kepada guru/pengawas, lalu memasukkannya melalui menu di pojok kanan atas aplikasi.
PIN Pengawas bersifat rahasia. Jangan pernah memberitahukan PIN Pengawas kepada siswa. Guru/pengawas wajib menginput PIN secara langsung di perangkat siswa untuk mencegah penyalahgunaan.

Ringkasan Penggunaan PIN:

  1. Mulai ujian pertama kali → gunakan Token / PIN Masuk (sekali pakai).
  2. Masuk kembali setelah keluar tidak sengaja → guru input PIN Pengawas di HP siswa.
  3. Selesai ujian dan ingin keluar → siswa minta PIN Keluar ke guru.

Examonlan secara otomatis mendeteksi aplikasi-aplikasi yang dapat digunakan untuk kecurangan. Jika aplikasi terlarang terdeteksi, ujian tidak dapat dimulai hingga aplikasi tersebut dinonaktifkan (disable) atau dihapus (uninstall).

Jenis Aplikasi Terlarang yang Sering Terdeteksi:

  • Floating / Melayang App: Game Booster, Game Turbo, Game Space bawaan HP, Screen Recorder melayang, Assistive Touch, aplikasi catatan melayang, dsb.
  • Emulator Android: BlueStacks, NoxPlayer, LDPlayer, MuMu Player, dan sejenisnya (biasanya dideteksi di PC/Laptop).
  • Aplikasi Kloning (Dual/Clone App): Dual Space, Parallel Space, aplikasi ganda bawaan HP (Dual Apps Xiaomi, Dual Messenger Samsung, dsb.).
  • Perekam Layar (Screen Recorder): Aplikasi perekam layar pihak ketiga maupun bawaan sistem yang sedang aktif.
  • Virtual Machine / VPN tertentu: Aplikasi yang dapat memanipulasi atau menyembunyikan aktivitas layar.

Cara Mengatasi:

  1. Nonaktifkan (Disable): Masuk ke Pengaturan → Aplikasi → cari aplikasi yang dimaksud → pilih Nonaktifkan / Disable. Cara ini tidak menghapus aplikasi secara permanen.
  2. Hapus (Uninstall): Jika tidak bisa dinonaktifkan, hapus aplikasi tersebut dari HP. Tekan lama ikon aplikasi → pilih Hapus / Uninstall.
  3. Tutup semua aplikasi yang berjalan di latar belakang melalui tombol Recent Apps, lalu bersihkan semuanya.
  4. Restart HP, kemudian buka kembali aplikasi Examonlan.
Jika setelah dinonaktifkan aplikasi masih terdeteksi, coba restart HP terlebih dahulu. Beberapa aplikasi sistem bawaan memerlukan restart agar perubahannya berlaku.

Jika Anda sudah mematikan NFC di pengaturan HP namun Examonlan masih mendeteksinya sebagai aktif, kemungkinan besar hal ini disebabkan oleh keterbatasan deteksi NFC pada tipe HP tertentu yang belum sepenuhnya didukung oleh versi aplikasi saat ini.

Yang Bisa Dicoba:

  1. Pastikan NFC benar-benar sudah dimatikan melalui Pengaturan → Koneksi → NFC (atau cari "NFC" di kolom pencarian pengaturan).
  2. Coba restart HP setelah mematikan NFC, kemudian buka kembali Examonlan.
  3. Pastikan aplikasi Examonlan sudah diperbarui ke versi terbaru melalui Google Play Store.
Jika masalah masih terjadi setelah mencoba langkah di atas, kemungkinan HP Anda belum didukung sepenuhnya untuk pendeteksian NFC. Tunggu pembaruan aplikasi berikutnya yang akan memperluas dukungan untuk lebih banyak tipe perangkat.
